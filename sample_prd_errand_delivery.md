# 跑腿配送系统 PRD

## 1. 文档信息

| 字段 | 内容 |
|---|---|
| 产品名称 | 同城跑腿配送系统 |
| 文档版本 | v1.0 |
| 适用端 | 用户端、骑手端、管理后台 |
| 编写目的 | 定义跑腿配送系统核心业务流程、功能范围、业务规则和验收标准，供研发、测试和运营使用 |

## 2. 背景与目标

### 2.1 背景

本地生活场景中，用户存在同城代买、代送、文件配送、生鲜配送、鲜花配送、帮办代办等即时服务需求。平台需要建设一套连接用户、骑手和平台运营人员的跑腿配送系统，实现订单发布、费用预估、智能派单、履约跟踪、支付结算和售后处理。

### 2.2 产品目标

1. 用户可以快速发布跑腿订单并实时查看配送进度。
2. 骑手可以接收订单、完成取件和送达操作。
3. 平台可以管理订单、骑手、费用规则、营销活动和异常订单。
4. 系统需要支持订单状态全链路流转和关键节点通知。
5. 系统需要支持基础风控，避免无效订单、重复支付和异常结算。

## 3. 用户角色

| 角色 | 描述 | 主要能力 |
|---|---|---|
| 普通用户 | 使用跑腿服务的 C 端用户 | 下单、支付、查看订单、取消订单、评价 |
| 骑手 | 承接配送任务的服务人员 | 接单、取件、配送、送达、异常上报 |
| 平台运营 | 平台后台运营人员 | 订单管理、骑手管理、费用配置、营销配置 |
| 平台财务 | 负责结算和账务处理的人员 | 查看流水、处理退款、骑手佣金结算 |
| 系统管理员 | 管理系统配置和权限 | 账号管理、角色权限、系统参数配置 |

## 4. 业务范围

### 4.1 本期包含

- 用户注册登录
- 地址管理
- 跑腿订单创建
- 配送费用预估
- 在线支付
- 系统派单与骑手抢单
- 骑手取件、配送、送达
- 订单取消与退款
- 订单评价
- 管理后台订单管理
- 骑手审核与状态管理
- 基础营销优惠券
- 财务流水和骑手佣金记录

### 4.2 本期不包含

- 跨城配送
- 多商户入驻管理
- 复杂路径规划算法
- 企业月结账户
- 第三方即时配送平台聚合调度

## 5. 核心业务流程

### 5.1 正常下单配送流程

```text
用户填写订单 → 系统计算费用 → 用户确认并支付 → 系统派单 → 骑手接单 → 骑手到达取件地 → 骑手确认取件 → 配送中 → 骑手确认送达 → 用户确认完成 → 系统结算佣金 → 用户评价
```

### 5.2 用户取消流程

```text
用户提交取消 → 系统判断订单状态 → 计算退款金额 → 取消订单 → 原路退款 → 通知用户/骑手
```

### 5.3 骑手异常上报流程

```text
骑手选择异常类型 → 上传说明和图片 → 系统冻结订单状态 → 平台运营介入处理 → 处理完成后继续配送/取消订单/退款
```

## 6. 功能需求

## 6.1 用户端

### 6.1.1 登录注册

#### 功能说明

用户可通过手机号验证码登录系统。首次登录自动创建用户账户。

#### 业务规则

1. 手机号必须为中国大陆 11 位手机号。
2. 验证码有效期为 5 分钟。
3. 同一手机号 60 秒内只能发送一次验证码。
4. 连续输错验证码 5 次后，该手机号 10 分钟内不可继续验证。

#### 验收标准

- 输入合法手机号后可以获取验证码。
- 验证码正确时登录成功。
- 验证码错误、过期、频繁发送时有明确提示。

### 6.1.2 地址管理

#### 功能说明

用户可以新增、编辑、删除常用地址，并设置默认地址。

#### 字段要求

| 字段 | 必填 | 说明 |
|---|---|---|
| 联系人 | 是 | 2-20 个字符 |
| 手机号 | 是 | 11 位手机号 |
| 所在城市 | 是 | 当前开通服务的城市 |
| 详细地址 | 是 | 门牌号、楼层等详细信息 |
| 经纬度 | 是 | 由地图选点生成 |
| 地址标签 | 否 | 家、公司、学校、其他 |

#### 业务规则

1. 用户最多保存 20 个常用地址。
2. 删除默认地址后，系统自动选择最近更新的地址为默认地址。
3. 未开通城市不可保存为服务地址。

### 6.1.3 创建跑腿订单

#### 功能说明

用户选择服务类型，填写取件地址、送达地址、物品信息、期望时间和备注后创建订单。

#### 服务类型

| 服务类型 | 描述 |
|---|---|
| 帮我送 | 用户已有物品，需要骑手配送到目标地址 |
| 帮我买 | 用户指定商品，需要骑手购买后送达 |
| 帮我取 | 用户指定地点取件并送达 |
| 帮办代办 | 用户委托骑手完成线下排队、打印、取号等事项 |

#### 订单字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 服务类型 | 是 | 帮我送/帮我买/帮我取/帮办代办 |
| 取件地址 | 是 | 帮我买时可为购买地点 |
| 送达地址 | 是 | 用户收货地址 |
| 物品名称 | 是 | 不超过 50 字 |
| 物品重量 | 是 | 单位 kg |
| 物品价值 | 否 | 用于保价和风控 |
| 期望取件时间 | 是 | 立即取件或预约时间 |
| 备注 | 否 | 不超过 200 字 |
| 联系电话 | 是 | 用户联系电话 |

#### 业务规则

1. 取件地址和送达地址必须在同一开通城市。
2. 默认最大配送距离为 30 公里。
3. 默认最大物品重量为 20kg。
4. 禁止配送违禁品，包括危险品、活体动物、管制刀具、违法物品等。
5. 预约订单最早可预约当前时间 30 分钟后，最晚可预约 7 天内。
6. 用户存在未支付订单时，可以继续创建订单，但未支付订单超过 15 分钟自动关闭。

#### 验收标准

- 必填字段缺失时不能提交。
- 地址超出服务范围时不能提交。
- 系统能根据地址、距离、重量和时间计算预估费用。
- 下单成功后生成唯一订单号。

### 6.1.4 配送费用预估

#### 功能说明

系统根据配送距离、重量、服务类型、时段和优惠券计算订单费用。

#### 计费规则

| 费用项 | 规则 |
|---|---|
| 起步价 | 3 公里内 12 元 |
| 距离费 | 超出 3 公里后，每公里 2 元，不足 1 公里按 1 公里计 |
| 重量费 | 超出 5kg 后，每 kg 1 元，不足 1kg 按 1kg 计 |
| 夜间服务费 | 22:00-次日 06:00 加收 5 元 |
| 恶劣天气费 | 平台配置，可加收 0-10 元 |
| 保价费 | 物品价值超过 500 元时，可选保价，按物品价值 1% 收取 |

#### 业务规则

1. 优惠券抵扣金额不得超过订单实付金额。
2. 费用预估结果有效期为 10 分钟。
3. 如果地址、重量、服务类型变化，需要重新计算费用。

### 6.1.5 在线支付

#### 功能说明

用户可使用微信支付或支付宝支付订单。

#### 业务规则

1. 订单创建后 15 分钟内未支付，订单自动关闭。
2. 支付成功后订单状态变为“待接单”。
3. 同一订单重复收到支付成功回调时，只处理一次。
4. 支付金额必须与系统订单金额一致，否则拒绝回调。

#### 异常场景

- 用户支付中关闭页面，再次进入订单详情时需要查询最新支付状态。
- 支付成功但回调延迟时，用户可手动刷新订单状态。
- 支付失败时订单保持“待支付”。

### 6.1.6 订单跟踪

#### 功能说明

用户可以查看订单当前状态、骑手信息、配送轨迹和预计送达时间。

#### 订单状态

| 状态 | 说明 |
|---|---|
| 待支付 | 订单已创建但未支付 |
| 已关闭 | 超时未支付或用户取消 |
| 待接单 | 已支付，等待骑手接单 |
| 待取件 | 骑手已接单，前往取件地 |
| 配送中 | 骑手已取件，正在配送 |
| 已送达 | 骑手确认送达 |
| 已完成 | 用户确认完成或系统自动完成 |
| 已取消 | 支付后取消成功 |
| 异常处理中 | 骑手或用户提交异常 |

#### 业务规则

1. 骑手接单后，用户可查看骑手姓名、手机号脱敏信息和当前位置。
2. 配送轨迹每 15 秒刷新一次。
3. 骑手确认送达后，如果用户 24 小时内未确认，系统自动完成订单。

### 6.1.7 订单取消与退款

#### 功能说明

用户在特定订单状态下可以取消订单并申请退款。

#### 取消规则

| 当前状态 | 是否可取消 | 退款规则 |
|---|---|---|
| 待支付 | 可取消 | 无需退款 |
| 待接单 | 可取消 | 全额退款 |
| 待取件 | 可取消 | 扣除 3 元调度费后退款 |
| 配送中 | 不可直接取消 | 需联系客服处理 |
| 已送达 | 不可取消 | 可发起售后 |
| 已完成 | 不可取消 | 可发起售后 |

#### 业务规则

1. 退款原路返回。
2. 退款成功后订单状态变为“已取消”。
3. 骑手已接单后取消，需要通知骑手。

### 6.1.8 评价反馈

#### 功能说明

订单完成后，用户可以对骑手服务进行评价。

#### 字段要求

| 字段 | 必填 | 说明 |
|---|---|---|
| 星级 | 是 | 1-5 星 |
| 评价标签 | 否 | 准时、态度好、包装完整、沟通顺畅等 |
| 文字评价 | 否 | 不超过 200 字 |
| 图片 | 否 | 最多 3 张 |

#### 业务规则

1. 每个订单只能评价一次。
2. 评价提交后不可修改。
3. 评价会影响骑手服务评分。

## 6.2 骑手端

### 6.2.1 骑手注册与审核

#### 功能说明

骑手提交个人资料和证件信息，平台审核通过后才可接单。

#### 字段要求

| 字段 | 必填 | 说明 |
|---|---|---|
| 姓名 | 是 | 与身份证一致 |
| 手机号 | 是 | 用于登录和联系 |
| 身份证号 | 是 | 18 位身份证号 |
| 身份证正反面照片 | 是 | 图片格式 jpg/png |
| 健康证 | 否 | 部分城市要求 |
| 交通工具类型 | 是 | 电动车/摩托车/汽车/步行 |
| 服务城市 | 是 | 已开通城市 |

#### 业务规则

1. 审核状态包括：待审核、审核通过、审核拒绝、已禁用。
2. 审核拒绝时必须填写拒绝原因。
3. 已禁用骑手不可登录接单。

### 6.2.2 接单

#### 功能说明

骑手可以查看附近待接订单并抢单，也可以接收系统派单。

#### 业务规则

1. 骑手必须处于“在线”状态才能接单。
2. 骑手同时最多持有 3 个未完成订单。
3. 距离取件地址超过 5 公里的订单默认不展示，除非平台开启远距离订单。
4. 同一订单只能被一个骑手成功接单。
5. 抢单成功后订单状态变为“待取件”。

### 6.2.3 取件

#### 功能说明

骑手到达取件地后确认取件。

#### 业务规则

1. 骑手需要上传取件凭证照片，帮我买订单需要上传购物小票。
2. 骑手距离取件地址超过 500 米时，系统提示“距离较远”，但允许继续操作并记录风险日志。
3. 取件成功后订单状态变为“配送中”。

### 6.2.4 送达

#### 功能说明

骑手到达送达地址后确认送达。

#### 业务规则

1. 骑手需要上传送达凭证照片或获取用户签收码。
2. 用户开启签收码时，骑手必须输入正确签收码才能确认送达。
3. 骑手距离送达地址超过 500 米时，系统提示“距离较远”，但允许继续操作并记录风险日志。
4. 送达成功后订单状态变为“已送达”。

### 6.2.5 异常上报

#### 功能说明

骑手配送过程中遇到问题时可以上报异常。

#### 异常类型

| 异常类型 | 说明 |
|---|---|
| 联系不上用户 | 电话无人接听或号码错误 |
| 地址错误 | 地址不存在或定位错误 |
| 物品不符 | 实际物品与订单描述不一致 |
| 用户拒收 | 用户拒绝接收物品 |
| 交通事故 | 骑手配送途中发生事故 |
| 其他 | 需要填写具体说明 |

#### 业务规则

1. 异常上报后订单状态变为“异常处理中”。
2. 平台运营处理前，用户和骑手都不可继续变更订单状态。
3. 异常处理结果包括继续配送、取消退款、订单完成、人工关闭。

## 6.3 管理后台

### 6.3.1 订单管理

#### 功能说明

运营人员可以查看、筛选、导出和处理订单。

#### 查询条件

- 订单号
- 用户手机号
- 骑手手机号
- 订单状态
- 服务类型
- 创建时间
- 支付时间
- 城市
- 是否异常

#### 操作能力

| 操作 | 适用状态 | 说明 |
|---|---|---|
| 查看详情 | 全部状态 | 查看订单全量信息和状态日志 |
| 指派骑手 | 待接单 | 手动指定骑手 |
| 取消订单 | 待接单/待取件/异常处理中 | 根据规则退款 |
| 标记完成 | 已送达/异常处理中 | 人工完成订单 |
| 导出订单 | 全部状态 | 导出 Excel |

### 6.3.2 骑手管理

#### 功能说明

运营人员可以审核骑手、查看骑手状态、禁用骑手和调整服务区域。

#### 业务规则

1. 禁用骑手前，如果骑手存在进行中订单，需要先完成或转派订单。
2. 骑手评分低于 3 分时，后台需要标记风险提醒。
3. 骑手近 7 天取消率超过 20% 时，后台需要标记风险提醒。

### 6.3.3 费用规则配置

#### 功能说明

平台可按城市配置计费规则。

#### 配置项

- 起步价
- 起步公里数
- 超出距离单价
- 免费重量
- 超重单价
- 夜间服务费
- 恶劣天气费
- 最高配送距离
- 最大物品重量

#### 业务规则

1. 费用规则修改后只影响新订单，不影响已创建订单。
2. 每个城市必须有一套启用中的费用规则。
3. 同一城市同一时间只能启用一套费用规则。

### 6.3.4 优惠券管理

#### 功能说明

平台可以创建满减券和折扣券。

#### 优惠券字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 优惠券名称 | 是 | 不超过 30 字 |
| 优惠类型 | 是 | 满减/折扣 |
| 使用门槛 | 是 | 满多少元可用 |
| 优惠金额/折扣 | 是 | 根据优惠类型决定 |
| 发放数量 | 是 | 大于 0 |
| 每人限领 | 是 | 大于 0 |
| 有效期 | 是 | 开始时间和结束时间 |
| 适用城市 | 否 | 为空表示全部城市 |

#### 业务规则

1. 已过期优惠券不可领取和使用。
2. 用户下单时只能使用一张优惠券。
3. 已使用优惠券的订单取消后，未过期优惠券可退回用户账户。

### 6.3.5 财务结算

#### 功能说明

系统记录用户支付流水、退款流水和骑手佣金流水。

#### 业务规则

1. 订单完成后生成骑手佣金记录。
2. 骑手佣金默认为订单配送费的 80%。
3. 骑手佣金进入“待结算”状态，T+1 可提现。
4. 订单退款后，需要冲正相关佣金记录。

## 9. 非功能需求

### 9.1 性能要求

1. 首页接口响应时间 P95 不超过 500ms。
2. 订单创建接口响应时间 P95 不超过 1s。
3. 费用预估接口响应时间 P95 不超过 800ms。
4. 订单列表查询支持 10 万级订单数据分页查询。
5. 骑手位置上报接口支持每 15 秒一次的高频上报。

### 9.2 可用性要求

1. 核心下单和支付链路月可用性不低于 99.9%。
2. 支付回调必须支持幂等处理。
3. 系统重启后，未完成订单状态不能丢失。

### 9.3 安全要求

1. 用户手机号、身份证号需要脱敏展示。
2. API 请求需要登录态校验。
3. 管理后台操作需要记录操作日志。
4. 支付回调需要验签。
5. 骑手证件图片需要限制访问权限。

### 9.4 兼容性要求

1. 用户端支持微信小程序和 H5。
2. 管理后台支持 Chrome、Edge 最近两个大版本。
3. 图片上传支持 jpg、png、jpeg，单张不超过 5MB。

## 10. 数据统计需求

平台后台需要提供以下统计指标：

1. 今日订单数
2. 今日支付金额
3. 今日完成订单数
4. 今日取消订单数
5. 订单完成率
6. 平均接单时长
7. 平均配送时长
8. 骑手在线人数
9. 异常订单数量
10. 优惠券使用金额

## 11. 验收场景示例

### 11.1 正常订单完成

```text
给定用户已登录且服务城市已开通
当用户填写合法取件地址、送达地址和物品信息并完成支付
且骑手成功接单、取件、送达
那么订单最终状态应为已完成
且系统应生成支付流水和骑手佣金记录
```

### 11.2 待接单取消

```text
给定用户已支付订单且订单状态为待接单
当用户发起取消订单
那么订单状态应变为已取消
且系统应创建全额退款记录
且订单不应再被骑手接单
```

### 11.3 骑手重复抢单

```text
给定订单状态为待接单
当两个骑手几乎同时抢同一订单
那么系统只允许一个骑手抢单成功
另一个骑手应收到订单已被抢的提示
且订单只绑定一个骑手
```

### 11.4 支付回调重复

```text
给定订单已收到一次支付成功回调
当支付平台再次发送同一支付单号的成功回调
那么系统不应重复更新订单金额
不应重复生成支付流水
不应重复触发派单
```

### 11.5 异常订单处理

```text
给定订单状态为配送中
当骑手上报联系不上用户异常
那么订单状态应变为异常处理中
且运营后台应出现待处理提醒
当运营选择继续配送
那么订单状态应恢复为配送中
```
