# 待办事项系统 PRD

## 1. 产品背景

用户在日常工作和学习中需要记录待办事项，并跟踪任务完成情况。系统提供一个简单的待办事项管理能力，帮助用户创建、查看、编辑、完成和删除任务。

## 2. 产品目标

1. 用户可以快速创建待办事项。
2. 用户可以查看所有待办事项。
3. 用户可以修改待办事项内容。
4. 用户可以标记待办事项为已完成。
5. 用户可以删除不需要的待办事项。

## 3. 用户角色

| 角色 | 描述 |
|---|---|
| 普通用户 | 使用系统管理个人待办事项的用户 |

## 4. 功能需求

### 4.1 创建待办事项

#### 功能说明

用户可以输入待办事项标题和描述，创建一条新的待办事项。

#### 字段要求

| 字段 | 必填 | 说明 |
|---|---|---|
| 标题 | 是 | 最多 50 个字符 |
| 描述 | 否 | 最多 200 个字符 |
| 截止日期 | 否 | 不能早于当前日期 |

#### 业务规则

1. 标题不能为空。
2. 标题超过 50 个字符时不能提交。
3. 创建成功后，待办事项默认状态为“未完成”。

### 4.2 查看待办事项列表

#### 功能说明

用户可以查看自己创建的所有待办事项。

#### 列表字段

| 字段 | 说明 |
|---|---|
| 标题 | 待办事项标题 |
| 状态 | 未完成/已完成 |
| 截止日期 | 用户设置的截止日期 |
| 创建时间 | 待办事项创建时间 |

#### 业务规则

1. 默认按创建时间倒序排列。
2. 用户可以筛选“全部”、“未完成”、“已完成”。
3. 当列表为空时，页面显示“暂无待办事项”。

### 4.3 编辑待办事项

#### 功能说明

用户可以修改待办事项的标题、描述和截止日期。

#### 业务规则

1. 已完成的待办事项仍然允许编辑。
2. 标题不能为空。
3. 修改成功后，列表展示最新内容。

### 4.4 完成待办事项

#### 功能说明

用户可以将未完成的待办事项标记为已完成。

#### 业务规则

1. 只有“未完成”状态的待办事项可以标记完成。
2. 标记完成后，系统记录完成时间。
3. 已完成事项不能重复标记完成。

### 4.5 删除待办事项

#### 功能说明

用户可以删除不再需要的待办事项。

#### 业务规则

1. 删除前需要二次确认。
2. 删除成功后，该待办事项不再出现在列表中。
3. 删除操作不可恢复。

## 5. 状态流转

```text
创建待办事项 → 未完成 → 标记完成 → 已完成
未完成/已完成 → 删除
```

## 6. 验收标准

### 6.1 创建待办事项

```text
给定用户打开待办事项页面
当用户输入合法标题并点击创建
那么系统应创建成功
并在列表中展示该待办事项
且状态为未完成
```

### 6.2 标题为空

```text
给定用户打开创建待办事项表单
当用户不输入标题并点击创建
那么系统不应提交
并提示“标题不能为空”
```

### 6.3 标记完成

```text
给定存在一条未完成的待办事项
当用户点击完成按钮
那么该事项状态应变为已完成
并记录完成时间
```

### 6.4 删除待办事项

```text
给定存在一条待办事项
当用户点击删除并确认
那么该事项应从列表中移除
```

## 7. 非功能需求

1. 页面操作响应时间不超过 1 秒。
2. 用户只能查看和操作自己的待办事项。
3. 系统应兼容 Chrome 和 Edge 浏览器最近两个大版本。
